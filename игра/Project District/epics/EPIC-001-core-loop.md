# EPIC-001: Базовый дневной цикл

## Description

Сделать минимальный скелет игры: старт, день, планирование, исполнение, отчет, следующий день.

## Stories

### STORY-001-001: Стартовая сессия
**User Story**: As a игрок, I want to начать новую сессию so that попасть в День 1 с готовым миром.
**Acceptance Criteria**:
- [x] Создается начальное состояние.
- [x] Текущий день равен 1.
- [x] Игрок видит утреннюю сводку.
**Size**: S

### STORY-001-002: Подтверждение дня
**User Story**: As a игрок, I want to подтвердить выбранные приказы so that день был рассчитан.
**Acceptance Criteria**:
- [x] Нельзя подтвердить невалидный приказ.
- [x] После подтверждения приказы блокируются.
- [x] Запускается turn engine.
**Size**: M

### STORY-001-003: Переход к следующему дню
**User Story**: As a игрок, I want to перейти к следующему дню so that продолжать кампанию.
**Acceptance Criteria**:
- [x] День увеличивается на 1.
- [x] Состояния временных эффектов обновляются.
- [x] Показывается новая утренняя сводка.
**Size**: S
