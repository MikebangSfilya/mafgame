# EPIC-006: Баланс и плейтест

## Description

После готового цикла проверить, работает ли игра на 4 районах, и настроить главные числа.

## Stories

### STORY-006-001: Балансные параметры
**User Story**: As a геймдизайнер, I want to менять числа so that быстро проверять гипотезы.
**Acceptance Criteria**:
- [ ] Доходы, защита, жара, шумность приказов вынесены в данные.
- [ ] Изменения не требуют переписывать resolver.
- [ ] Есть стартовый пресет 0.1.
**Size**: M

### STORY-006-002: Плейтест 20 дней
**User Story**: As a разработчик, I want to пройти 20 дней so that проверить темп и драму.
**Acceptance Criteria**:
- [ ] Зафиксирован лог 20 дней.
- [ ] Отмечены скучные, слишком жесткие и слишком легкие места.
- [ ] Есть список правок баланса.
**Size**: S

### STORY-006-003: Критерий "еще один день"
**User Story**: As a команда, I want to оценить интерес so that решить, расширять ли игру.
**Acceptance Criteria**:
- [ ] Игрок минимум 3 раза менял план после отчета.
- [ ] Игрок минимум 1 раз явно боялся высокой жары.
- [ ] Игрок хотел сделать следующий ход после неприятного исхода.
**Size**: S
